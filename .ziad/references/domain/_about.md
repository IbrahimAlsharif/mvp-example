# References — domain/

Domain-calibrated reference material. These files cover the same topics as
`analytic-methods/` and `workflows/` but are tuned to the specific client
context defined in `.ziad/domain.yaml`.

## What belongs here

- Alert taxonomy and severity tiers as calibrated for this client
- Briefing standards specific to this client's output format
- Entity taxonomy and disambiguation rules for this domain
- Source credibility framework tuned to this client's source landscape
- Crisis monitoring workflows adapted for this client's priorities
- Any generic concept (intelligence cycle, narrative tracking, etc.) that has
  been specifically calibrated to this client's operational context

## Recommended files to create

Adapt these to your domain — not all will be relevant:

| File | What it covers |
|------|---------------|
| `alert-taxonomy.md` | Alert types and severity tiers |
| `briefing-standards.md` | Briefing format and quality bar |
| `entity-taxonomy.md` | Entity types and disambiguation rules |
| `source-credibility.md` | Source evaluation for this domain's sources |
| `crisis-monitoring.md` | Crisis monitoring workflow |
| `index.md` | Index of all domain reference files |

## When to write here

After the generic reference (e.g., `analytic-methods/source-evaluation-frameworks.md`)
proves insufficient for the client's specific context and needs to be
re-calibrated.
