Findings

F1 · risk — floor fake_affordance/dead_navigation + cog-walkthrough Q2. On an empty timeline the quick-add rail is not rendered (CosmicCommandCenter.tsx:155); the placeholder instructs "press the timeline" but nothing is clickable. J2's own primary entry is a first-use dead-end. Fix: render the rail when empty, or make the placeholder itself open quick-add.
F2 · risk — WCAG 2.1.1 Keyboard / 4.1.2. The rail is a div with only mouse handlers (role=null, tabindex=null, CosmicTimeline.tsx:282). The whole quick-add entry is mouse-only — keyboard/SR users cannot open it. Fix: a focusable rail with a key-to-add, or a visible "add at date" control.
F3 · risk — Heuristic 9 / confirmation correctness. Future-dated events are labelled "اليوم/today": relativeHint maps days <= 0 → today (EventCard.tsx:32, date.ts:13). Misreports the date the user just chose on the load-bearing position↔date mapping. Fix: handle days < 0 distinctly; never call a future date "today".
F4 · risk (re-test for blocking) — HIGHEST IMPACT — floor occluded_or_offscreen_control (390px). The absolutely-positioned 588px-tall popup lands at y≈511, so Save sits at y≈1052–1085, below the 844px viewport, and the page does not scroll to it (scrollHeight===844, scrollTo is a no-op). Reachable only via an inner scroll/zoom → recorded risk, not a hard block. Fix: centered scrollable bottom-sheet on ≤480px.
F5 · risk — Method 4 legibility (craft threshold). Dialog type is systematically <14px: note 12px; time/place/labels/Save label 11px (QuickAddPopup.tsx:169,185,204,216,282); Save is 33px tall (<44px). Worst for the elderly variant. Fix: ≥14px body, ≥44px Save.
F6 · risk — WCAG 2.1.2/2.4.3/4.1.2 + Heuristic 3/4. QuickAddPopup role=dialog has no aria-modal, no initial focus (stays on BODY), no focus trap, and ESC does not close it (verified). Its sibling EventModal does all of this correctly → an internal inconsistency. Fix: match the EventModal dialog standard.
F7 · risk — floor container_overflow (390px). Rail renders 443px wide in a 390px viewport (clipped ~82px); the compressed 256px rail makes taps imprecise (a test tap for empty space opened an existing node instead). Header export + sign-out render off-screen (same class as J1 F3). Fix: constrain/scroll the rail; menu the header at ≤480px.
F8 · advisory — RTL convention / Heuristic 2. Axis runs past-left → future-right and headers agree (future on the right) — internally consistent, but inverts the RTL reading-order expectation. journey.md's "past=right" is stale. Advisory: confirm intended direction with design; the good HoverReadout disambiguates either way. Not a code bug.
F9 · advisory — Method 5 gradient_text/ai_color_palette tell. Save button gradient fill + gradient brand wordmark (from J1 F10). Passes contrast; identity tell only. Fix: solid brand color.
